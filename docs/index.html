Changelog

v1.3.1 更新公告

UI 优化、状态修复、模型扩展和 Yan Computer Use 插件在本次快速更新中完成

v1.3.12026-07-20

这次更新，不只是换了一个版本号

  • v1.3.1 快速更新 — 优化核心页面 UI，修复模型异常时任务状态和通知不同步的问题，扩展模型配置，并加入新的 Yan Computer Use 插件。
  • Yan Project Map — 把目录、文件、符号和依赖边变成可以缩放和探索的地图。你可以先看全局结构，再沿着入口节点定位到需要修改的模块。
  • Yanxi Code 双向协同 — Yan Agent 会检测已安装的 IDE，并以带回执的协议交接工作区。Yanxi Code 无论冷启动还是已经运行，都会刷新到当前工作区，而不是停在欢迎页。
  • 内置终端与浏览器 — 命令执行、网页预览和结果验证都留在任务上下文旁边。Agent 可以边读代码边运行真实检查，把“应该可以”变成可复核的证据。
  • 桌面宠物与移动端 — 桌面宠物跟随当前任务，显示运行状态和资源占用，并提供停止入口。移动端可以搜索、切换和管理任务，模型和生成图片会与桌面端同步。
  • 多模态工作流 — 图片入口会根据模型能力动态出现，视觉模型可以处理截图和界面问题。图片生成、编辑、预览和下载都保留在任务历史中，刷新后仍可继续使用。
  • 更开放的模型目录 — 新增 Agnes、自定义模型，并加入百川、Yi、腾讯混元和硅基流动。自定义模型只需填写 Base URL、API Key 与模型 ID，即可直接连接任意 OpenAI 兼容服务，不依赖本地路由或模型映射。
  • 更可靠的任务交付 — 五个任务拥有独立状态、工作区、中止控制和 MCP 工具快照。写入后回读、完成门控和文件变更摘要让每一步都更容易检查。

Capabilities

Agent 能做什么

每一项能力都围绕一个问题：如何让真实工作更快、更清楚、更可恢复

五任务并行

每个任务拥有独立的 agentState、工作区、中止控制和 MCP 快照。切换任务只改变你正在看的内容，后台任务不会被打断。

终端与浏览器

Agent 可以在内置终端执行命令，在内置浏览器打开本地页面或验证结果。执行和观察留在同一工作区里，减少来回切换带来的遗漏。

Yan Project Map

本地索引负责事实，地图负责让事实变得可浏览。你可以沿着符号和依赖关系定位问题，再把相关节点交给 Agent 深入处理。

可靠编辑与验证

精确编辑要求旧文本唯一匹配，写入之后还会回读确认。Todo 完成门控和文件变更摘要让你能快速判断结果是否真的落地。

多模态与图片

图片输入会按模型能力开启，视觉模型可以理解截图、界面和设计稿。生成或编辑的图片会被保留，可从桌面端和移动端继续查看。

Skill 与 MCP

Skill 把团队方法沉淀为可复用的任务入口，MCP 则把浏览器、桌面和外部服务接进来。每个任务保留自己的工具快照，复杂工作也能并行推进。

Workflow

从想法到结果

你说目标，Agent 负责把中间过程变成可见、可验证的步骤

1

描述目标

用自然语言告诉 Agent 你想交付什么。无需先知道应该改哪个文件。

2

看见计划

Agent 自动拆分 Todo 和验收条件。每一步都能在右侧面板里跟踪。

3

调用工具

读取、编辑、执行、浏览和分析按需组合。失败会被分类处理，而不是默默跳过。

4

验证结果

写后回读、运行检查并记录文件变更。完成时你拿到的是结果和证据，不只是回答。
